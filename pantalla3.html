3. Servicios y notificaciones en App

Tareas en segundo plano, notificaciones, sincronización y cómputo en la nube

Tareas en segundo plano

Las tareas en segundo plano permiten que una aplicación siga ejecutando procesos aun cuando el usuario no la tiene abierta. Estas tareas son esenciales para sincronización de datos, servicios de ubicación, descargas, reproducción de música, notificaciones push, etc.

Tipos según plataforma

Android Services — procesos continuos en background
WorkManager — tareas programadas (sincronización)
Foreground Services — tareas largas con notificación visible
iOS Background Tasks — ejecución limitada para IA, ubicación o descargas
Service Workers (Web/PWA) — cache, sincronización, push

En aplicaciones web o PWAs, los Service Workers son la base para implementar tareas offline, push notifications y sincronización en segundo plano.

Más info en MDN: Service Worker API

Notificaciones Push

Las notificaciones permiten comunicarse con el usuario incluso cuando la app está cerrada. En Android e iOS se usan servicios nativos (FCM y APNs). En la Web se combinan:

  • Push API — recibe mensajes desde un servidor.
  • Notifications API — muestra alertas al usuario.
  • Service Worker — escucha eventos push en background.

Guía completa: MDN Push API

Plataformas más usadas

Firebase Cloud Messaging (FCM) — Android, iOS y Web
Apple Push Notification Service (APNs) — iOS nativo
OneSignal — Multi-plataforma, fácil integración
Expo Push Notifications — apps con React Native

Proceso profesional de implementación

  1. Registrar un Service Worker (PWA) o servicio nativo (Android/iOS).
  2. Solicitar permisos de notificación al usuario.
  3. Obtener el token del dispositivo (identificador único).
  4. Enviar el token al servidor para guardarlo.
  5. Configurar proveedor (FCM, APNs, OneSignal).
  6. Recibir notificaciones en background y manejar eventos.
  7. Respuestas a acciones (botones de “Aceptar”, “Abrir”, etc.).

Las notificaciones pueden incluir imágenes, botones, acciones rápidas y deep links hacia pantallas específicas.

Servicios en la nube relacionados

Firebase — autenticación, base de datos, hosting, FCM
AWS Amplify / Lambda — microservicios y funciones serverless
Azure Mobile Apps — APIs, notificaciones y sincronización
Supabase — base de datos, auth y real-time open-source

Los servicios en la nube permiten delegar tareas intensivas como procesamiento de datos, análisis de actividad, envío de notificaciones masivas o actualización de información en tiempo real.

Ejemplo: flujo de entrega de una notificación

  1. La app obtiene el token del dispositivo y lo envía a un servidor.
  2. El servidor solicita a Firebase Cloud Messaging enviar un mensaje.
  3. FCM entrega el mensaje al dispositivo objetivo.
  4. El sistema operativo muestra la notificación en la barra.
  5. El usuario la toca → la app se abre y procesa la acción.
